SLIDE 7: How AI Actually Reads Your Prompts
--------------------------------------------
Title: How AI Actually Reads Your Prompts

Subtitle: Think of AI like a really smart intern—helpful, but needs clear direction

Main Content:

What AI is good at:
✓ Following specific instructions
✓ Using examples you provide
✓ Filling in reasonable defaults
✓ Working in phases/steps

What AI struggles with:
✗ Reading your mind
✗ Knowing your unstated preferences
✗ Understanding vague requests like "make it better"
✗ Handling massive requests all at once

Key Insight Box:
"AI doesn't know what 'professional' means to YOU. It needs specifics: colors, layout type, sections, tone, who it's for."

Bottom Text:
The clearer you are, the less back-and-forth fixing you'll do.


SLIDE 8: The 4-Part Prompting Framework
----------------------------------------
Title: The 4-Part Prompting Framework

Subtitle: This is what pros use (and what we'll practice today)

Four columns with clear examples:

1. CONTEXT
   Set the stage - who/what/why
   Example: "You're building a website for a fitness coach who works with busy parents. The goal is to get people to book discovery calls."

2. INSTRUCTIONS
   What you want built
   Example: "Create a landing page with: hero section, 3 program options, before/after testimonials, booking calendar, FAQ section."

3. CONTENT
   The actual information
   Example: 
   Coach name: Sarah Mitchell
   Tagline: 'Get fit in 20 minutes a day—no gym required'
   Programs: Home Workout Plan ($99/mo), Nutrition Coaching ($149/mo), Full Package ($199/mo)

4. FORMAT
   How it should look/behave
   Example: "Modern, energetic design. Primary color: bright orange (#FF6B35). Mobile-first layout. Big, clear CTA buttons."

Bottom Banner:
Use all 4 parts = Better results in fewer tries


SLIDE 9: Breaking Big Ideas Into Phases
----------------------------------------
Title: Don't Build Everything at Once

Subtitle: The #1 tip from Reddit, Twitter, and every platform: Break it into phases

Why This Matters:

When you ask for everything at once, AI can:
✗ Get confused and mix up sections
✗ Time out or hit token limits
✗ Create inconsistent code
✗ Be harder to debug

The Better Way - Phase-by-Phase:

Phase 1: Core Structure
"Build the homepage with header, hero section, and footer. Use placeholder text."
✓ Quick to build
✓ Easy to review
✓ Solid foundation

Phase 2: Add Content
"Now add the About section with my bio and the Services section with 3 service cards."

Phase 3: Features
"Add a contact form that sends to my email and a testimonials carousel."

Phase 4: Polish
"Update colors to match my brand: primary #DC2626, secondary #991B1B. Add smooth scroll animations."

Bottom Text (Large):
Each phase takes 2-5 minutes. Total time? Same. Quality? Way better.


SLIDE 10: Common Prompting Mistakes (And How to Fix Them)
-----------------------------------------------------------
Title: Common Mistakes Beginners Make

Format: Table with 3 columns: ❌ Mistake | Why It Fails | ✅ Better Approach

Row 1:
❌ "Make it look professional"
→ Too vague—AI guesses
✅ "Use clean layout, sans-serif fonts, white space, business blue (#1E40AF)"

Row 2:
❌ "Add a contact form"
→ Missing specs
✅ "Add contact form with Name, Email, Message fields. Send to hello@example.com. Show success message after submit."

Row 3:
❌ One giant paragraph of every detail
→ AI loses track
✅ Break into Context, Instructions, Content, Format

Row 4:
❌ "Fix it" or "Make it better"
→ AI doesn't know what's wrong
✅ "The header overlaps the hero image on mobile. Fix the spacing and make header sticky."

Row 5:
❌ Building everything in one prompt
→ Errors multiply, hard to debug
✅ Phase 1: Structure. Phase 2: Content. Phase 3: Features. Phase 4: Polish.

Bottom Banner:
Being specific isn't being picky—it's being effective.


SLIDE 11: Platform Differences (What Works Where)
---------------------------------------------------
Title: Not All Platforms Work the Same Way

Subtitle: Here's when to use each tool and how to prompt them

Four platform cards:

🎨 LOVABLE (What we used in Class 1)
Best for: Beginners, visual apps, quick websites
Prompting Tips:
• Very descriptive about UI/design
• Mention color codes, layout types
• Ask for Supabase if you need a database
• Start simple, iterate with chat
Example prompt: "Build a recipe sharing app. Grid of recipe cards, each shows image, title, cooking time. Click card to see full recipe. Use warm orange theme (#FF6B35)."

⚡ BOLT.NEW
Best for: Full-stack apps, prototypes with backend
Prompting Tips:
• Specify both frontend AND backend needs
• Use "prompt enhance" feature
• Break into discussion mode phases
• Mention frameworks if you have preferences
Example prompt: "Build a simple task manager. Frontend: list of tasks with checkboxes, add new task form. Backend: save tasks to database, mark complete/incomplete. Clean, minimal design."

💻 CURSOR
Best for: Developers, editing existing code, complex projects
Prompting Tips:
• Use @file and @folder to give context
• Ask questions before generating code
• Request it to act as code reviewer
• Keep context windows small
Example prompt: "@src/components/Header.tsx - Review this header component. Check for accessibility issues, performance problems, and suggest improvements. Then refactor with your recommendations."

🚀 REPLIT
Best for: Learning, collaborating, full dev environment
Prompting Tips:
• Ask for initial structure, then iterate
• Use for open-ended projects
• Great for debugging with AI
• Generous free tier for practice
Example prompt: "Create a Python Flask app that displays a random motivational quote on the homepage. Include a 'Get New Quote' button. Use simple Bootstrap styling."

Bottom Text:
For today's practice: We'll stick with Lovable (you already have accounts). But now you know when to try others.

SLIDE 15: Common Problems & Quick Fixes
----------------------------------------
Title: When Things Go Wrong (And They Will)

Subtitle: Here's how to debug like a pro

Problem-Solution Grid:

Problem 1: AI picks wrong colors
Quick Fix: "Change primary color to #DC2626 and secondary to #991B1B throughout the site."

Problem 2: Layout broken on mobile
Quick Fix: "The header overlaps content on mobile screens. Fix spacing and make it responsive."

Problem 3: Missing a section you mentioned
Quick Fix: "Add the testimonials section below the services section. Include 3 testimonial cards with name, photo, and quote."

Problem 4: Text is placeholder/lorem ipsum
Quick Fix: "Replace all placeholder text with: [paste your actual content]"

Problem 5: AI times out or fails
Quick Fix: Start a new chat, use a shorter prompt, build in smaller phases.

Pro Tip Box:
"Keep a doc of your working prompts. When something works well, save it as a template for next time."

Bottom Banner:
Debugging is part of the process. Every error teaches you how to prompt better next time.


SLIDE 16: Advanced Tips (For When You're Ready)
-------------------------------------------------
Title: Level Up: Advanced Prompting Techniques

Subtitle: Try these once you've got the basics down

Four Advanced Techniques:

1. Role-Playing
"Act as a senior UX designer. Review this layout and suggest 3 improvements for better user flow."
Why it works: AI adjusts its perspective and gives more specialized feedback.

2. Examples-First
"Here's a screenshot of a design I like [attach]. Build something similar but for a fitness app instead of a restaurant."
Why it works: Visual references reduce ambiguity.

3. Version Control
Use Git or version history. Before making big changes, commit/save your current version.
Why it works: You can always roll back if AI changes too much at once.

4. The Question-First Approach
"Before building, ask me 5 questions about this project to make sure you understand what I need."
Why it works: Catches misunderstandings before code is written.

Bottom Text:
You don't need these today—but they're here when you're ready to level up.
